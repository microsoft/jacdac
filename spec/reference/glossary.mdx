## Glossary

Below are a list of some JACDAC and GitHub specific terms we use across our sites and documentation.

### device

A device is any hardware that exposes one or more services with JACDAC.

### module

A module is a device that typically exposes one service, and is meant to be assembled together with other modules to create a large device.
A module is a building block of a device.

### service

### register

An addressed data location on a **service**, that may be get, set or streamed. 

### event

### command

### packet

### field

A data field that represents a part of the data from a **register**

### bus

A network of JACDAC devices

### firmware identifier

Unique identifier identifiying a class of devices. Can be queried via the [control service](/services/control).

### service class

Unique identifier identifiying a type of service. The list of service is announced by a device.

### Service index 

Index in the list of services of a device. Not to be confused with _service class_.
